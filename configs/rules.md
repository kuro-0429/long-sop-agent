# 交付规范

> 本文件定义文件结构、打包、截图、录屏的交付要求。技术约束见 base.md，生成流程见 generator.md。

---

## 一、文件结构（严格）

**单页面滚动型（A01 架构）：** 可用长截图一张截全 → 用 `preview.png`
```
fd_XXX.zip
└── fd_XXX/
    ├── prompt.md
    ├── preview.png          ← 单张长截图
    ├── src/
    │   └── index.html
    └── video.mp4
```

**SPA / 多面板型（A02–A09 架构）：** 无法滚动截全，每个 panel/页面单独截图 → 用 `preview/` 文件夹
```
fd_XXX.zip
└── fd_XXX/
    ├── prompt.md
    ├── preview/             ← 文件夹，每个面板一张
    │   ├── preview_01.png
    │   ├── preview_02.png
    │   └── ...
    ├── src/
    │   └── index.html
    └── video.mp4
```

**判断规则：**
| 架构 | 截图方式 | 文件形式 |
|------|---------|---------|
| A01 单页滚动 | GoFullPage 长截图 | `preview.png` |
| A02 视差滚动 | GoFullPage 长截图 | `preview.png` |
| A03 标签页切换 | 每个 Tab 截一张 | `preview/` |
| A04 圆形展开 SPA | 每个 Panel 截一张 | `preview/` |
| A05 幻灯片 | 每张幻灯片截一张 | `preview/` |
| A06 滚动吸附 | 每个吸附区截一张 | `preview/` |
| A07 水平轨道 | 每个 Panel 截一张 | `preview/` |
| A08 竖向抽屉 | 每个抽屉状态截一张 | `preview/` |
| A09 3D翻页书 | 每页正反面截一张 | `preview/` |

**命名规则：**
- 统一前缀 `fd`，三位数字编号，从 001 递增
- zip 文件名必须与文件夹名完全一致
- `src/` 目录下只允许 `index.html`，不允许其他文件

**严禁出现：**
- `.pen` 文件
- `node_modules` / `package.json` / `.git`
- `__MACOSX` / `.DS_Store` / `Thumbs.db` 等系统文件

## 二、prompt.md 格式

```
## Round 1

[角色定义 + 设计系统 + 页面区块结构，每区块 ≥ 8 行]

## Round 2

[功能性交互系统 — Modal / Accordion / Toast / Tab 逐一写明触发+行为+关闭+ARIA]

## Round 3

[响应式断点（含 390px）+ 可访问性 + 动效规范]

## Round 4

[打磨细节 + 完整 Acceptance Criteria 验收清单]
```

- **最少 4 轮（强制）**：结构 → 功能交互 → 响应式 → 打磨验收
- **全篇总字符数 ≥ 5500**（安全线）；优秀区间 8000~12000；< 4000 铁定不通过
- **交互行为描述 ≥ 40%，视觉描述 ≤ 40%，结构+验收 ≥ 20%**
- 每个页面区块最少 8~12 行描述（布局 / 内容 / 视觉 / 交互 / 响应式）
- **Round 4 必须包含逐条可验证的 Acceptance Criteria 清单**（功能验收 ≥ 10 项）
- 只写给 AI 的 Prompt，不包含 AI 的回复
- 全英文（中英均可，强烈推荐英文）
- Round 1 之前**绝对不能有任何内容**（无标题行、无品牌信息、无说明文字）

## 三、截图规范

**滚动页（A01/A02）— 单张长截图 `preview.png`：**

| 要求 | 规格 |
|------|------|
| 类型 | 长截图（完整页面，滚动截图） |
| 格式 | PNG |
| 宽度 | 建议 1920px |
| 浏览器 UI | 不允许出现标签栏、地址栏、书签栏 |
| 鼠标指针 | 尽量隐藏 |
| 动画状态 | 所有动画必须处于最终可见状态（非初始 opacity:0 状态） |
| 数字计数器 | 必须显示最终值，不能是 0 |
| 图片 | 全部正常加载，无裂图 |

推荐工具：GoFullPage（Chrome 扩展）

**SPA / 多面板（A03–A09）— 多张截图放入 `preview/` 文件夹：**

| 要求 | 规格 |
|------|------|
| 每张截图 | 当前面板/状态的全屏截图 |
| 命名 | `preview_01.png`、`preview_02.png`……依次编号 |
| 格式 | PNG，建议 1920×1080 |
| 顺序 | 按用户浏览顺序排列（Panel 1 → Panel N） |
| 浏览器 UI | 同上，不允许出现 |
| 动画状态 | 每张截图须等页面动画播完再截 |
| 最少张数 | 每个主要 Panel/状态至少一张，建议截全部 |

## 四、录屏规范

| 参数 | 要求 |
|------|------|
| 格式 | MP4 |
| 帧率 | **24 FPS（严格）** |
| 时长 | 4–60 秒 |
| 录制范围 | 全屏（F11 全屏模式） |
| 分辨率 | 建议 1920×1080 |

**必须展示：**
1. 从顶部开始缓慢滚动展示整页
2. 鼠标悬停各按钮（hover/active 效果）
3. 悬停卡片（上浮/阴影变化）
4. 悬停导航链接
5. 点击可交互按钮（Tab切换/FAQ/轮播等）
6. 触发所有动画（滚动渐入/计数器等）
7. 最后滚回顶部

**严禁出现：**
- 浏览器标签栏 / 地址栏 / 书签栏
- 系统通知弹窗
- 任务栏
- 其他无关窗口

**帧率转换（FFmpeg）：**
```bash
# 转换为 24fps
ffmpeg -i input.mp4 -r 24 -c:v libx264 -pix_fmt yuv420p output.mp4

# WebM 转 MP4 24fps
ffmpeg -i input.webm -c:v libx264 -r 24 -pix_fmt yuv420p output.mp4

# 2倍速压缩时长
ffmpeg -i input.mp4 -filter:v "setpts=0.5*PTS" -r 24 output_2x.mp4

# 批量转换
for f in *.mp4; do ffmpeg -i "$f" -r 24 -c:v libx264 -pix_fmt yuv420p "24fps_$f"; done
```

## 六、常见驳回原因速查

| # | 驳回原因 | 如何避免 |
|---|---------|---------|
| 1 | **Prompt 偏弱，验收条件没写清楚** | Round 4 必须有完整 Acceptance Criteria 清单；每区块 ≥ 8 行描述 |
| 2 | **功能交互缺位（纯展示页）** | Prompt 中逐条写明 Modal/Accordion/Toast/Tab 的触发元素、内容、关闭方式、ARIA |
| 3 | **交互维度单一（只有 scroll reveal / hover 变色）** | 点击后必须有反馈链路：触发→反馈→后续操作，≥ 8 种不同交互类型 |
| 4 | **Prompt 轮次不足** | 必须 **4 轮**：结构 → 功能交互 → 响应式 → 打磨验收 |
| 5 | **Prompt 字数不足** | 总字符数 ≥ 5500；禁止重复内容/空洞词填充 |
| 6 | 图片加载失败 | 只用 Unsplash URL，提交前验证 |
| 7 | 文件命名不规范 | 严格 `fd_xxx` 格式 |
| 8 | 视频帧率不对 | FFmpeg 转 24fps |
| 9 | 网页类别不准确 | 如实填写实际内容 |
| 10 | 不是长截图 | GoFullPage 完整截图 |
| 11 | 录屏出现浏览器 UI | F11 全屏录制 |
| 12 | 代码需要编译 | 只用原生 HTML/CSS/JS |
| 13 | 风格单一重复 | 每天涵盖多种类型和风格 |

## 八、特效与交互密度要求（强制 · 核心！）

> **核心原则：页面必须特效酷炫，交互设计几乎随处可见，给人以极强的视觉冲击力和沉浸式体验。**
> **判断标准：用户打开页面 1 秒内必须能看到至少一个"哇"级别的视觉效果。如果需要仔细找才能注意到，那就不够。**

---

### 8.1 什么是特效，什么不是特效（必须理清）

**特效 = 让用户在视觉上产生「哇」的反应的动画/视觉表现。**

✅ 以下才算特效：
- 140+个粒子在 Canvas 上漂浮，鼠标靠近时粒子被推开并连线
- 文字中每个字符从 rotateX(90°) + blur(8px) 翻转进入，逐字延迟 50ms
- 点击按钮时爆发出 40+ 个不同形状粒子，向四周扩散带旋转和重力下沉
- 背景有模糊的紫色 blob 在缓慢浮动（translateX/Y 循环 + scale + blur:60px）
- 350px 直径的渐变光球以 lerp 0.08 的插值速度缓慢追踪鼠标

❌ 以下不算特效（不计入特效数量）：
- opacity:0→1 淡入（太基础，只是过渡）
- 按钮 hover 变色（CSS 入门级）
- 卡片 hover 上浮 4px + 阴影加深（只是"有 hover 状态"）
- Scroll Reveal 淡入、导航栏滚动变色、汉堡菜单展开收起、Toast 弹窗、返回顶部按钮

---

### 8.2 效果数值校准表（严格执行）

> **之前所有效果平淡的根本原因：数值太保守。以下是强制最低标准。**

| 属性 | ❌ 禁止值（看不见） | ✅ 最低要求值（惊艳） |
|------|---------------------|----------------------|
| 光晕/光球透明度 | 5% | **≥ 15%**（15-20%） |
| 卡片3D倾斜角度 | ±5deg | **≥ ±15deg**（推荐 ±20deg） |
| 粒子爆发数量 | 12个 | **≥ 40个**（推荐 40-60） |
| Canvas 背景粒子数 | 30个 | **≥ 100个**（推荐 100-200） |
| 粒子大小 | 固定3px | **3-10px 随机** |
| 粒子扩散距离 | 40px | **60-120px** |
| 光标圆环 hover 尺寸 | 跟默认差不多 | **从40px→60px 明显变化** |
| 粒子动画持续时间 | 0.3s | **≥ 0.9s** |
| 彩纸/礼花持续时间 | 0.3s | **≥ 1.8s** |
| 文字入场 blur 值 | 无 blur | **blur(8px) → blur(0)** |
| translateY 入场距离 | 20px | **≥ 40px** |
| 元素 hover 缩放 | 1→1.02 | **1→1.05-1.15** |
| box-shadow 发光 | 几乎不可见 | **0 0 30px rgba(color, 0.35)** |
| 光球/光晕直径 | 100px | **≥ 350px** |
| lerp 插值系数（光球跟随） | 无延迟 | **0.08-0.12**（必须有拖尾果冻感） |
| perspective 值 | 无 | **600-800px** |

---

### 8.3 七大用户动作 × 必须响应的效果（逐一落实）

> **每个页面必须覆盖下表中至少 5 种用户动作类型，每种动作至少实现 2 个效果。**

#### 🖱️ 动作 1：鼠标在页面上移动（mousemove）— 至少选 2 个

| 效果 | 实现要求 |
|------|----------|
| **自定义光标** | `cursor:none`，小圆点(8px) 精确跟随 + 大圆环(40px, 2px border) 带 lerp 0.12 缓冲跟随，圆环必须有**明显拖尾感** |
| **鼠标光晕/光球** | ≥ 350px 直径 radial-gradient，**透明度 ≥ 15%**，lerp 0.08 跟随，pointer-events:none |
| **Canvas 粒子排斥** | Hero 区 100-200 个粒子，鼠标 < 150px 时施加反向力，粒子间距 < 120px 时连线 |
| **墨水晕染** | 鼠标经过区域背景色 radial-gradient 扩散 → 2s 消退 |
| **鼠标拖尾轨迹** | 每 50ms 在鼠标位置生成小圆，0.5s 后消失，形成彩色轨迹 |

#### 🖱️ 动作 2：鼠标悬停元素（mouseenter/mousemove on element）

**悬停按钮 — 以下至少实现 2 个：**
| 效果 | 实现要求 |
|------|----------|
| **光标变形** | 圆环从 40px→60px，添加 background rgba |
| **磁吸效果** | `translate(offsetX*0.3, offsetY*0.3)`，鼠标离开时弹性弹回 |
| **发光增强** | box-shadow 从 0 0 20px 到 0 0 40px，透明度 0.2→0.4 |
| **液态填充** | ::before height 从 0→100%，颜色从底部灌满 |
| **边框描绘** | SVG rect stroke-dashoffset 顺时针画一圈 |

**悬停卡片 — 以下至少实现 2 个：**
| 效果 | 实现要求 |
|------|----------|
| **3D 透视倾斜** | `perspective(600px) rotateX(y*-20deg) rotateY(x*20deg)`，**最大倾斜 ≥ ±15deg** |
| **内部光效跟随** | radial-gradient overlay 圆心跟随鼠标位置实时更新 |
| **图片/内容缩放** | scale(1.05)，transition 0.5s |
| **卡片翻转** | preserve-3d，正面产品图，背面详情，hover 时 rotateY(180deg) |
| **霓虹流动边框** | conic-gradient 旋转边框，@property --angle 动画 |

**悬停标题文字 — 以下至少实现 1 个：**
| 效果 | 实现要求 |
|------|----------|
| **字符波浪涟漪** | 拆为单个 span，逐字 translateY(-10px) 回弹，delay = 索引 × 30ms |
| **光带扫过(Shimmer)** | background-clip:text，background-position 从 200% 到 -200% |
| **Glitch 抖动** | ::before ::after RGB 位移(cyan+magenta)，clip-path 随机跳动 |

#### 🖱️ 动作 3：点击（click）— 必须有强烈视觉反馈

| 效果 | 实现要求 |
|------|----------|
| **按钮压缩弹回** | scale(0.95) → 弹回，transition 0.15s |
| **粒子爆发** | ≥ 40 个粒子，随机大小(3-10px)、随机形状(圆/方/棱)、随机颜色(6种)、随机角度(0-360°)、随机速度(60-120px)，带旋转+消失(0.9s) |
| **彩纸爆炸** | ≥ 50 个彩纸，混合形状(圆+长方形)，带重力(先上飞再下落)，旋转(±540°)，1.8s |
| **墨水涟漪** | 点击处圆形从 0→200px 的墨色扩散圈 |
| **闪光+震屏** | 全页 filter:brightness(1.5) 闪白 0.05s + body translateX(±3px) 震动 2-3 次 |
| **金色光晕扩散** | 金色圆环从 0→300px 扩散消失(0.8s)，涟漪效果 |

#### 🖱️ 动作 4：拖拽（mousedown + mousemove）— 适用时必须实现

| 效果 | 实现要求 |
|------|----------|
| **Before/After 滑块** | 分割线跟随 + clip-path 实时裁切 + 48px 圆形手柄带**脉冲发光动画**(2s循环) |
| **可拖拽卡片排序** | 被拖卡片 opacity:0.5 + scale(0.95)，放置区域实线+变色 |
| **Tinder 滑动** | 左滑删除右滑收藏 |

#### ⌨️ 动作 5：键盘输入（focus/input）— 表单区域必须实现

| 效果 | 实现要求 |
|------|----------|
| **浮动标签** | placeholder 上浮到边缘，translateY + scale:0.78，颜色变主题色 |
| **边框发光** | border-color 变主题色 + box-shadow 外发光，**整个输入框在发光** |
| **实时验证** | 输入超阈值时出现绿色 ✓（scale(0)→1 弹性），删减时弹性缩回 |

#### 📜 动作 6：滚动（scroll）— 以下至少实现 3 个

| 效果 | 实现要求 |
|------|----------|
| **阅读进度条** | 顶部 3px 渐变色(primary→accent→gold)，宽度 = scrollY/(scrollHeight-innerHeight)×100% |
| **导航栏变化** | 滚动 > 50px 时 backdrop-filter:blur(20px) + 底部阴影 |
| **元素入场** | IntersectionObserver + translateY(40px) + opacity:0→1（基础设施，不计入特效数量） |
| **视差效果** | 前景/中景/远景不同滚动速度 |
| **背景色渐变过渡** | 不同 section 间背景色平滑过渡 |
| **数字计数** | rAF + easeOutExpo + 视口触发，从 0 滚到目标值 |

#### 📜 动作 7：页面首次加载（DOMContentLoaded）— 必须有惊艳入场

| 效果 | 实现要求 |
|------|----------|
| **Hero 标题 3D 入场** | 拆为单字符 span，初始 opacity:0 + rotateX(90deg) + translateY(40px) + blur(8px)，逐字 delay=索引×50ms |
| **副标题延迟淡入** | 延迟 1.2s，opacity:0→1 + translateY:20→0 |
| **按钮区延迟淡入** | 延迟 1.5s |
| **Hero 图片入场** | 延迟 0.5s，scale(0.9)+rotate(-3deg) → 正常 |
| **Canvas/动态背景启动** | requestAnimationFrame 立即启动 |

---

### 8.4 动画密度硬指标

- 页面中**每个可见区域**都必须包含至少一种动画效果
- **禁止出现"静态区域"**：任何区块在视口内停留 2 秒内必须有可感知的视觉运动
- CSS keyframe 动画总数 ≥ 10 组（不含 mandatory keyframes）
- `transition` 属性覆盖率 ≥ 90%（所有可交互元素必须有过渡）

### 8.5 交互设计密度硬指标

- 可观测到的**不同交互反馈类型 ≥ 15 种**（含 hover、click、drag、scroll、input、toggle、swipe、keyboard 等维度）
- 大型交互组件 ≥ 2 个，每个组件内部交互步骤 ≥ 3 步
- 所有按钮、卡片、链接、输入框必须有**完整四态**（default / hover / active / focus-visible）
- **hover 效果必须多维度联动**：至少同时改变 2 项以上属性（位移+阴影、缩放+边框、旋转+发光等）

### 8.6 大型视觉特效要求（至少满足 4 项）

- [ ] Canvas 粒子/星空/波浪等全屏动态背景
- [ ] 自定义光标系统（跟随 + hover 变形 + 拖尾/混合模式）
- [ ] CSS 3D 变换（卡片倾斜 ≥±15deg / 翻转 / 景深效果）
- [ ] 视差滚动或多层视差效果
- [ ] SVG 路径动画或 morphing（stroke-dashoffset 描绘）
- [ ] 全局性装饰动画（浮动 blob blur:60px+ / 发光脉冲 / 渐变流动）
- [ ] 文字排印动效（逐字 3D 飞入 / 打字机 / 乱码解码 / 分裂重组）
- [ ] 滚动驱动进度指示器（进度条 / 粒子轨迹 / 形态变化）
- [ ] 点击粒子爆发（≥40粒子）或彩纸爆炸（≥50彩纸）
- [ ] 磁吸按钮（translate 跟随鼠标偏移 + 弹性弹回）

### 8.7 交互反馈链路要求

- 每个用户操作都必须有**即时视觉反馈**（≤ 100ms）
- 点击按钮必须有 ripple / flash / scale / 粒子爆发 等效果
- 关键操作必须有**链式反馈**：触发 → 中间状态 → 完成状态
- Toast 通知、状态切换、进度展示等反馈必须有入场/退场动画

### 8.8 效果评级参考

| 级别 | 描述 | 例子 |
|------|------|------|
| ❌ 不合格 | 只有基础 CSS hover、淡入淡出 | 按钮变色、scroll reveal |
| ⚠️ 合格但不出彩 | 有交互但不惊艳 | Tab 切换、手风琴、基础 modal |
| ✅ 达标 | 让人"哦~"的效果 | 3D 倾斜卡片、倒计时翻牌、拖拽排序 |
| 🌟 优秀 | 让人"哇！"的效果 | 彩纸爆炸、文字乱码解码、磁性按钮 |
| 💎 炸裂 | 第一次看会截图发同事 | 鼠标光球追踪、clip-path 揭示、字符3D飞入 |

**每个页面至少 12 个 ✅ 级以上的交互特效，其中至少 3-4 个 🌟/💎 级别。**

### 8.9 禁止事项

- ❌ 任何区域出现纯静态展示（无任何动画或交互）
- ❌ hover 只变颜色不变形态
- ❌ 点击无反馈的按钮或链接
- ❌ 生硬的状态切换（无过渡动画）
- ❌ 单调重复的动画（所有入场动画用同一种 translateY+opacity）
- ❌ 光晕透明度 < 10%（等于看不见）
- ❌ 卡片倾斜角度 < 10deg（肉眼分辨不出）
- ❌ 粒子爆发数量 < 30 个（跟没有一样）
- ❌ 所有效果都做"最低可行"版本（每个效果必须做到用户一定能注意到的程度）
- ❌ 每个网站都用同一套效果组合

## 九、特效风格多元化要求（强制）

> **严禁每个网站都用同一套效果！每个网站必须选择不同的特效风格方案，让每个页面打开后第一眼感觉就不一样。**

### 9.1 八大风格方案（每个网站选一个主方案，可从其他方案借 1-2 个效果混搭）

**方案 A「粒子宇宙」** — 适合科技 / SaaS / 太空
- 背景：Canvas 粒子网格（100-200粒子+连线+鼠标排斥）
- 光标：圆点+圆环 + 大光晕(15%)
- 按钮：磁吸 + 光标放大
- 卡片：3D perspective 倾斜(±20°) + 内部光效
- 点击：40粒子爆发(圆/方/棱)
- 文字：rotateX(90°)+blur 逐字翻转入场
- 滚动：渐变进度条 + 视差背景

**方案 B「水墨流动」** — 适合日式 / 文艺 / 极简
- 背景：SVG 波浪线底部波动（3层不同速度+透明度），无 Canvas
- 光标：无自定义光标，但鼠标经过区域墨水晕染(radial-gradient 扩散→2s消退)
- 按钮：边框描绘（SVG stroke-dashoffset 顺时针画一圈）
- 卡片：纸张掀起（rotateY(8deg)+单边阴影），非3D倾斜
- 点击：墨水涟漪（墨色圆形从0→200px扩散），非粒子
- 文字：SVG 手写描绘（stroke-dashoffset 100%→0%）
- 滚动：背景颜色随滚动位置平滑过渡（暖白→淡青→浅灰）

**方案 C「霓虹赛博」** — 适合游戏 / 电竞 / 音乐
- 背景：conic-gradient 极光旋转 + 网格线(1px, 5%透明度)
- 光标：十字准星 + 拖尾彩色轨迹(每50ms生成小圆，0.5s消失)
- 按钮：Glitch 抖动（::before ::after RGB位移 cyan+magenta）
- 卡片：霓虹流动边框（conic-gradient @property --angle 旋转）
- 点击：闪光+震屏（brightness(1.5) 0.05s + translateX(±3px) 震动）
- 文字：Glitch 故障（随机字符闪烁→逐个替换为正确字符）
- 滚动：clip-path:polygon 斜切分隔 + 视差

**方案 D「自然呼吸」** — 适合环保 / 有机 / 健康
- 背景：3个大圆 blur:80px CSS 浮动 blob，无 Canvas
- 光标：无自定义光标，但图片随鼠标微倾(±5°, 像植物向光)
- 按钮：液态填充（::before height 0→100%，文字色反转）
- 卡片：四角长出叶子（clip-path, hover 时 scale(0→1)+rotate）
- 点击：种子发芽（绿色线上长+顶部展开叶子，CSS animation）
- 文字：逐字从 blur(10px)+scale(1.2) → 清晰，像雾中浮现
- 滚动：飘落叶子/花瓣（4-6片，各自旋转+左右飘摆）

**方案 E「奢华立体」** — 适合时尚 / 珠宝 / 高端品牌
- 背景：纯色/渐变 + 大型 radial-gradient 光斑随滚动位移
- 光标：金色实心圆(6px) + 500px 聚光灯（mix-blend-mode:lighten）
- 按钮：金色边框从左到右画出（gradient background-size 0%→100%）
- 卡片：翻转（preserve-3d，正面产品，背面详情，rotateY(180deg)）
- 点击：金色圆环涟漪（0→300px 扩散消失 0.8s）
- 文字：逐字符从金色→正常文字色（像金箔揭开）
- 滚动：大型图片 clip-path:inset reveal（幕布拉开效果）

**方案 F「几何构成」** — 适合建筑 / 设计 / 教育
- 背景：CSS 网格线 + 交叉点小圆点
- 光标：靠近的网格交叉点圆点变大变亮（JS 距离计算）
- 按钮：border-radius 从 4px→50%（方变圆），颜色填充
- 卡片：内容"拆解"（图片/标题/描述各向不同方向偏移5-10px，爆炸视图感）
- 点击：几何碎片（20个三角/正方/圆形 clip-path 向外飞散）
- 文字：字符从四面八方(translateX/Y ±200px+rotate随机) 飞到正确位置
- 滚动：SVG 几何图形(三角/六边形)不规则分隔线

**方案 G「复古打字机」** — 适合文学 / 咖啡 / 手工
- 背景：纸张纹理(CSS SVG filter噪点)，微黄底色
- 光标：无自定义光标，墨水印跟随(200px radial-gradient, sepia色调)
- 按钮：打字键效果（translateY(2px)下陷 + Neo-Brutalist偏移阴影）
- 卡片：纸张角折叠（CSS gradient 折角，hover 0→30px）
- 点击：邮戳盖章（圆形 dashed border + rotate 歪斜 + 淡出）
- 文字：打字机逐字出现(60ms/字) + 闪烁光标 `|`(3秒后消失)
- 滚动：侧边手绘风格 SVG 进度指示器

**方案 H「液态流体」** — 适合饮料 / 化妆品 / 香水
- 背景：3个大型 CSS blob（@keyframes 缓慢变换 border-radius），半透明叠加
- 光标：鼠标位置影响 blob 偏移方向（CSS变量联动 translateX/Y）
- 按钮：波纹涟漪（radial-gradient 从 hover 点 0→100% 扩展）
- 卡片：液态渐变边框(linear-gradient background-position旋转) + backdrop-filter
- 点击：水滴溅射（大小不一圆形+椭圆+飞溅点，弹性 overshoot 动画）
- 文字：逐字符 scaleY(0→1)，origin:bottom，像从水面浮起
- 滚动：SVG 液态波浪分隔线 + 持续波动动画

### 9.2 风格选择规则

1. **每个网站选择 1 个主方案**，可从其他方案**借 1-2 个效果**混搭
2. **同一批次的网站（如连续生成的 7-8 条），不能有 2 个选同一个主方案**
3. **方案选择必须与网站主题相关**：科技→A，日料→B，游戏→C，有机→D，奢侈品→E，建筑→F，咖啡→G，香水→H
4. **鼓励创造新的第 9、10 个方案**，不限于以上 8 种
5. 关键是让每个网站打开后**第一眼感觉就不一样**：背景不同、光标不同、hover 效果不同、点击效果不同
6. **在 prompt.md 的 Round 1 中必须标注所选风格方案**（如"风格方案：A 粒子宇宙 + E 金色涟漪借用"）

## 十、全元素四态交互完整性（强制）

> **核心原则：页面上每一个可见的元素都不是"摆设"。每个按钮、卡片、板块、图片、文字区域都必须有完整的四态交互 + 酷炫特效 + 交互后的逻辑响应。用户碰到页面上的任何东西都应该感到"这个网站是活的"。**

### 10.1 四态定义与强制标准

**所有可见元素必须具备以下四个状态，且状态之间必须有明显的视觉差异和动画过渡：**

| 状态 | 定义 | 最低视觉要求 |
|------|------|-------------|
| **Default（默认态）** | 元素未被交互时的静息状态 | 必须已经有视觉存在感（不能是纯灰色/纯白色无特征块）；建议有微动画（呼吸脉冲、渐变流动、轻微浮动等）让元素看起来"活着" |
| **Hover（悬停态）** | 鼠标移入元素区域 | 必须至少同时改变 **3 项** 视觉属性（如：位移 + 阴影 + 发光、缩放 + 边框色 + 内部光效、倾斜 + 阴影偏移 + 图片缩放）。**禁止只变颜色** |
| **Active（激活态/点击态）** | 鼠标按下的瞬间 | 必须有**即时物理反馈**：scale 压缩(0.95-0.97) / translateY 下沉(2-3px) / 短暂闪光 / 涟漪扩散。反馈必须在 **≤ 50ms** 内出现 |
| **Focus-visible（键盘焦点态）** | Tab 键聚焦时 | 必须有明显的 focus ring（不能用 `outline:none` 后不补任何可见替代）；推荐：主题色 box-shadow 外发光 + border 变色 |

### 10.2 各元素类型的具体要求

#### 🔘 按钮（Button / CTA / Link-button）

| 状态 | 必须实现的效果 |
|------|---------------|
| Default | 有微妙的发光/渐变流动/边框动画，不能是纯色死板按钮 |
| Hover | 磁吸偏移 / 液态填充 / 发光增强 / 光标变形 / 边框描绘 — 至少 2 种联动 |
| Active | scale(0.95) 压缩 + ripple 涟漪 / 粒子爆发 / 闪光，**≤ 50ms 触发** |
| Focus-visible | 主题色 box-shadow 2px 外扩发光 + border 变色 |
| **交互后逻辑** | 点击后必须触发**有意义的后续**：打开 Modal / 切换内容 / Toast 通知 / 添加到购物车 / 表单提交 / 导航跳转 / 状态切换。**禁止点击后什么都不发生** |

#### 🃏 卡片（Card / Product Card / Feature Card / Testimonial）

| 状态 | 必须实现的效果 |
|------|---------------|
| Default | 有投影 + 边框/渐变，建议微浮动动画 |
| Hover | 3D 透视倾斜(≥±15deg) / 内部光效跟随鼠标 / 图片缩放(1.05) / 边框发光 / 内容位移 — 至少 3 种联动 |
| Active | 整体 scale(0.98) 下压 + 阴影收缩 + 短暂亮度变化 |
| Focus-visible | 边框高亮 + 外发光 |
| **交互后逻辑** | 点击后必须有**内容展开/详情展示**：打开 Modal 详情页 / 翻转露出更多信息 / 手风琴展开描述 / 跳转到对应 section / 灯箱放大。**禁止卡片只有 hover 无点击逻辑** |

#### 🖼️ 图片（Image / Gallery Item / Hero Image / Product Photo）

| 状态 | 必须实现的效果 |
|------|---------------|
| Default | 有入场动画（scale/fade/reveal），建议有 Ken Burns 微缩放或微视差 |
| Hover | clip-path 揭示(circle/inset) / 饱和度从灰→彩 / 3D 倾斜 / 缩放(1.05-1.1) + 渐变 overlay 出现 + 文字/按钮淡入覆盖 — 至少 2 种联动 |
| Active | 短暂亮度变化 + 边框闪烁 |
| Focus-visible | 可见 focus ring（对于可交互图片） |
| **交互后逻辑** | 点击后必须：打开灯箱全屏查看 / 弹出产品详情 / 图片画廊左右切换 / Before/After 对比。**禁止图片不可点击（纯装饰小图标除外）** |

#### 📦 板块/区域（Section / Feature Block / Hero / Footer）

| 状态 | 必须实现的效果 |
|------|---------------|
| Default | 有背景动画（粒子/blob/渐变流动/波浪），不能是纯色静态背景 |
| Hover/进入 | 鼠标进入板块时有**全局级响应**：背景粒子速度变化 / 光标样式切换 / 背景色微调 / 装饰元素反应 |
| Active | 板块内的可交互元素响应点击 |
| Scroll 进入 | 每个板块滚动进入视口时必须有**独特的入场动画**（不能每个 section 都用相同的 fadeInUp，必须差异化：左滑入/右滑入/缩放弹入/翻转入/模糊聚焦等） |
| **交互后逻辑** | 板块内必须包含至少 1 个可操作的交互功能（Tab 切换 / 手风琴 / 滑块 / 计算器 / 过滤器 / 评分 / 计数器等），**禁止纯展示性板块（只有标题+文字+图片、无任何可操作内容）** |

#### 🔗 链接/导航项（Nav Link / Footer Link / Inline Link）

| 状态 | 必须实现的效果 |
|------|---------------|
| Default | 文字有区分度（颜色或下划线暗示可点击） |
| Hover | 下划线从中间展开 / 光带扫过 / 字符波浪 / 背景色滑入 — 至少 1 种动效 |
| Active | 颜色/亮度瞬间变化 |
| Focus-visible | 可见 focus 指示 |

#### 📊 数据/信息元素（Counter / Progress / Badge / Tag / Stat）

| 状态 | 必须实现的效果 |
|------|---------------|
| Default | 滚动进入时有入场动画（计数滚动 / 进度条填充 / badge 弹入） |
| Hover | 放大(1.1) + 发光 / Tooltip 弹出详细信息 |
| Active | 可点击的数据元素必须有 ripple |
| **交互后逻辑** | 如果是可交互的（如 filter tag），点击后必须有**状态切换逻辑**（选中/取消 + 关联内容过滤/更新） |

### 10.3 交互后逻辑的硬性要求

> **仅有视觉反馈是不够的。每个可交互元素点击后必须触发有意义的业务逻辑。**

| 元素 | 必须有的交互后逻辑 |
|------|-------------------|
| CTA 主按钮 | → 打开 Modal / 触发表单 / 显示 Toast 确认 / 页面内锚点跳转 |
| "Learn More" 按钮 | → 展开更多内容 / 打开详情 Modal / 滚动到详情 section |
| 产品卡片 | → 打开产品详情 Modal / 翻转显示背面信息 / 添加到购物车 + 购物车 badge 跳动 |
| 图片 | → 灯箱全屏查看 / 画廊切换 / 弹出信息覆盖层 |
| FAQ 条目 | → 手风琴展开答案（smooth height 动画） |
| Tab 按钮 | → 切换内容面板 + 滑动指示线 |
| 评分星星 | → 点亮到对应星级 + 发光 + 显示感谢文字 |
| 社交媒体图标 | → hover 放大+发光 + 点击打开对应链接（或 Toast 提示） |
| 价格方案 | → 月/年切换 + 价格数字动画过渡 |
| 表单提交 | → 验证 + 彩纸爆炸 + Toast 成功通知 + 表单清空 |

**禁止出现的情况：**
- ❌ 按钮只有 hover 效果，点击后什么都不发生
- ❌ 卡片只能看不能点
- ❌ 图片纯静态展示（大型图片必须可点击查看）
- ❌ 任何看起来可交互的元素（有 hover 状态的），实际点击无响应
- ❌ `href="#"` 或 `onclick=""` 空操作

## 十一、交互融合主页面原则（强制）

> **核心原则：所有交互功能必须有机融入主页面设计，成为页面内容的一部分。禁止创建与页面主题/内容脱节的独立交互区域。**

### 11.1 禁止的做法

以下做法**严格禁止**：

| ❌ 禁止的模式 | 具体表现 | 为什么禁止 |
|--------------|---------|-----------|
| **独立小游戏区域** | 在页面某处嵌入一个小游戏（打地鼠、拼图、贪吃蛇等），与页面主题无关 | 与品牌/产品内容脱节，破坏页面专业性 |
| **独立互动玩具** | 单独一块区域放一个 Canvas 画板 / 弹球模拟 / 刮刮卡，与其他内容无关联 | 像是硬塞进去的演示 demo |
| **独立工具小部件** | 嵌入与主题无关的计算器 / 转换器 / 代码编辑器 | 用户会觉得困惑"这跟页面有什么关系" |
| **Demo 展示区** | 专门开一个区域"看看这些酷炫特效"，里面放一堆不相干的动画效果 | 特效应该融入内容，不是被"展览" |
| **独立全屏弹窗游戏** | 点击某按钮弹出全屏的小游戏或互动体验，与主页面完全脱节 | 打断用户浏览流程，与页面目的无关 |

### 11.2 正确的做法

以下做法**鼓励并推荐**：

| ✅ 正确的模式 | 具体表现 | 为什么好 |
|--------------|---------|---------|
| **产品图片 → 灯箱详情** | 点击产品图片，弹出 Modal 显示大图 + 详细信息 + 相关推荐 | 是页面内容的自然延伸 |
| **卡片 → 翻转/展开详情** | hover 或点击卡片，展开更多内容或翻转到背面 | 交互服务于内容呈现 |
| **产品配置器融入产品区** | 在产品展示区域直接选择颜色/尺码，联动预览图变化 | 交互与产品展示融为一体 |
| **评分系统嵌入评价区** | 在 Testimonial 区域内，用户可以点击星级评分 | 交互是评价内容的一部分 |
| **Before/After 对比融入产品区** | 用拖拽滑块展示产品效果前后对比 | 交互服务于产品展示 |
| **价格方案切换** | 月/年 Toggle 切换，价格卡片实时更新 | 交互是定价内容的一部分 |
| **FAQ 手风琴** | 问答区直接展开/收起 | 交互是内容组织方式 |
| **购物车流程** | 添加商品 → 购物车 badge 更新 → Toast 确认 | 交互是电商逻辑的一部分 |
| **时间线/步骤展示** | 点击时间线节点展开详情 | 交互服务于信息层级 |
| **画廊过滤器** | 点击标签过滤展示不同类别的作品 | 交互是内容筛选的一部分 |
| **表单 → 提交 → 反馈** | 联系表单填写 + 验证 + 提交 + 成功反馈 | 交互是页面转化目标 |

### 11.3 判断标准

在设计每个交互功能时，问自己以下三个问题：

1. **去掉这个交互，页面内容是否完整？**
   - 如果答案是"完全不受影响" → ❌ 这个交互可能是硬塞进去的
   - 如果答案是"缺少这个交互，用户无法完成某个内容操作" → ✅ 这是有机融合的

2. **这个交互用到的数据/内容，与页面主题有关吗？**
   - 打地鼠游戏里的地鼠与咖啡品牌无关 → ❌
   - 产品对比滑块里展示的是本品牌产品对比 → ✅

3. **用户在使用这个交互时，是否仍然在浏览页面内容？**
   - 用户进入了一个完全独立的"游戏模式"，要"退出"才能继续浏览 → ❌
   - 用户点击卡片弹出 Modal，关闭后继续浏览下一个 section → ✅

### 11.4 特殊说明

以下情况**不受此规则限制**：

- **Modal / Lightbox / Side Panel**：由页面元素触发、展示页面相关内容的弹窗是允许的且被鼓励的
- **主题相关的互动量身定制**：如酒店网站做一个"选择入住日期"的日历交互，虽然是"独立组件"但与页面主题强相关，属于✅
- **游戏化元素但与主题融合**：如运动品牌网站做一个"拖拽搭配运动装备"的配置器，虽然有游戏感但服务于产品展示，属于✅
